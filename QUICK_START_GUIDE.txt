QUICK START GUIDE - Happy Father's Day Web App
================================================

STEP 1: GET YOUR FIREBASE CREDENTIALS (5 minutes)
===================================================

1. Go to https://firebase.google.com
2. Click "Go to console"
3. Create a new project (or use existing)
4. Enable Firestore Database (Start in test mode)
5. Go to Settings (gear icon) > Project settings
6. Find "Your apps" section > Click "web" icon
7. Copy the firebaseConfig object

Your config will look like:
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project",
  storageBucket: "your-project.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcd1234",
  measurementId: "G-XXXXX"
};


STEP 2: UPDATE HTML FILES (2 minutes)
======================================

Open index.html in text editor:
1. Find the firebaseConfig section (around line 387)
2. Replace all YOUR_*_HERE placeholders with your actual Firebase config values
3. Save the file

Open view.html in text editor:
1. Find the firebaseConfig section (around line 327)
2. Replace all YOUR_*_HERE placeholders with your actual Firebase config values
3. Save the file


STEP 3: ADD YOUR LOGO (1 minute)
=================================

1. Prepare your logo image (recommended: 512x512 pixels, PNG format)
2. Name it: pixelforge-logo.png
3. Place it in the same folder as index.html and view.html


STEP 4: SET UP FIRESTORE SECURITY RULES (2 minutes)
=====================================================

Go to Firestore Database in Firebase Console:
1. Click "Rules" tab
2. Replace the entire content with:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /wishes/{docId} {
      allow read: if true;
      allow create: if request.resource.size < 1000000;
      allow update: if false;
      allow delete: if false;
    }
  }
}

3. Click "Publish"


STEP 5: TEST LOCALLY (5 minutes)
==================================

1. Open index.html in your web browser
2. Try uploading a photo
3. Enter your name and message
4. Click "Done"
5. Click "Send Love"
6. Copy the generated link
7. Open it in a new tab - you should see the envelope

If it works, you're ready to deploy!


STEP 6: DEPLOY TO VERCEL (recommended - 3 minutes)
===================================================

Option A: Using Vercel CLI
1. Install: npm install -g vercel
2. In project folder: vercel deploy
3. Follow prompts
4. Your app will be live!

Option B: Using Vercel Web Interface
1. Go to vercel.com
2. Sign in with GitHub account
3. Click "New Project"
4. Select your GitHub repository
5. Click "Deploy"

Option C: Deploy to Netlify
1. Go to netlify.com
2. Click "New site from Git"
3. Select your GitHub repository
4. Click "Deploy site"


STEP 7: SET UP GITHUB REPOSITORY (3 minutes)
==============================================

1. Create new repository on GitHub
2. Clone to your computer
3. Add all files to the folder:
   - index.html

This is only safe because the keys are environment-specific.
In a real app with multiple environments, use environment variables.

TROUBLESHOOTING
================

Q: Firebase connection fails
A: Check your Firebase config credentials are correct
   Ensure Firestore is enabled in Firebase console

Q: Photos don't upload
A: Check browser console for errors
   Ensure file size is under 5MB
   Try different image format (JPG, PNG)

Q: Link sharing doesn't work
A: Ensure view.html is on same domain as index.html
   Check that the document ID in URL is correct

Q: Animations are slow
A: Works best on modern browsers (Chrome, Firefox, Safari, Edge)
   Disable other browser extensions if they slow things down

Q: Firebase storage quota exceeded
A: Upgrade Firebase plan
   Or delete old test documents from Firestore


SECURITY NOTES
==============

Your Firebase configuration is visible in the HTML because:
- It's public/client-side configuration (designed to be exposed)
- Security comes from Firestore rules (we set to read-only)
- API keys are restricted to your project only
- Real security depends on the Firestore rules

Never expose your Firebase Admin SDK keys!
